# Project Language

This file keeps shared project vocabulary focused, so Bruno and future agents can speak in the same terms faster and more reliably.

Product terms are the main purpose. Add workflow terms only when they shape future behavior. Challenge new or conflicting terms against this file before adding them, then update it only when the resolved meaning is durable enough to help future work.

## Product Terms

- **Personal website**: Bruno Jardon's public website at `bjardon.com`.
- **Selected work**: Curated projects that signal the right capabilities to clients and recruiters; not a complete project archive.
- **Current scope**: The active implementation contract for what the project is building now. Scope direction changes require matching repo-doc updates.
- **Source-controlled content**: Site content that lives in repo-managed files and changes through the normal repo workflow, not through an external CMS.
- **Web-rendered resume**: Resume content rendered as a website page from repo-owned source content.
- **Unpublished page**: A route or page that may exist in the codebase but is not ready for public discovery; keep it out of navigation and sitemap until ready.
- **Indexable presence**: A public, crawlable site intended to rank for name search, especially "Bruno Jardon" and "Bruno Jardón".

## Operating Terms

- **Repo source of truth**: Repo docs are authoritative for current project direction until changed.
- **Upstream planning source**: External planning context that may inform future repo updates but is not authoritative until reflected in repo docs.
- **Issue Tracker**: Tactical tracking for issues, slices, and delivery status. It is not the source of truth for product direction.
- **Agentic engineering workflow**: Using agents for intake, planning, implementation, and documentation while keeping authority and decisions in the repo.
- **Interactive agent session**: A run where Bruno is actively present, prompting, reviewing changes, and can provide local context such as an existing localhost URL.
- **Unattended agent run**: A run where the agent is expected to proceed without Bruno actively watching or supplying local runtime context, such as cloud or background execution.
