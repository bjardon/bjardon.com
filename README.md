# Personal Website

This is my personal website: a public home for who I am, what I build, selected work, my resume, and future services.

## Audience

The primary audience is prospective freelance or contract clients evaluating me for engineering work.

The secondary audience is recruiters and hiring managers who need a sharper, more complete read than a profile page alone.

## Current Scope

The current iteration should establish a focused, indexable presence at `bjardon.com` with:

- Homepage with a focused introduction and navigation.
- Projects page with curated selected work and GitHub links.
- Resume page rendered on the web from current source content.
- Unpublished services page, kept out of navigation and sitemap until ready.

## Constraints

- Keep hosting cheap or near-free.
- Keep content source-controlled.
- Do not make this a CMS-driven content site.
- Published pages should use real content.
- Unfinished or unpublished pages stay out of navigation and sitemap until ready.
- Contact surfaces are links only; no contact form in the current scope.
- English only in the current scope.
- No PDF resume in the current scope.

## Working Agreement

- Repo docs are the source of truth for current project direction.
- External planning sources may inform future repo updates, but are not authoritative until reflected here.
- If an external source conflicts with repo docs, flag the mismatch, propose a resolution, and ask before changing scope or implementation.
- Scope direction changes should update the relevant repo docs as part of the same change or immediately before it.
- Avoid duplicating durable guidance across files; let one doc own the detail and link to it from elsewhere.
