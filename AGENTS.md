# Agent Guidance

- Read `README.md` first for project purpose, current scope, repo map, and working agreement.
- Read `LANGUAGE.md` before naming concepts; challenge conflicts before adding terms.
- Use `plans/` for durable planning artifacts when they help, not as a required gate.
- Flag external-source conflicts loudly, propose a resolution, and ask before changing scope or implementation.
- Keep product-shaping dependencies and sensitive surfaces ask-first: hosting, domains, production, billing, analytics, secrets, and contact endpoints.
- Do not stage, commit, push, or deploy unless asked.
- Add architecture notes or ADRs only when the project has real system shape or hard-to-reverse decisions.
